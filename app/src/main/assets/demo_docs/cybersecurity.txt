Кибербезопасность

Кибербезопасность - защита компьютерных систем, сетей и данных от цифровых атак, несанкционированного доступа и повреждения.

Основные принципы безопасности

CIA Triad (Триада ЦДК)
Три основных принципа информационной безопасности:

Confidentiality (Конфиденциальность)
Защита информации от несанкционированного доступа. Только авторизованные пользователи могут получить доступ к данным.

Методы: шифрование, контроль доступа, аутентификация.

Integrity (Целостность)
Гарантия того, что данные не были изменены несанкционированно. Данные остаются точными и полными.

Методы: хеширование, цифровые подписи, контрольные суммы.

Availability (Доступность)
Обеспечение доступа к информации и системам для авторизованных пользователей когда это необходимо.

Методы: резервное копирование, защита от DDoS, отказоустойчивость.

Типы кибератак

Malware (Вредоносное ПО)
Программы, созданные для нанесения вреда:

Virus (Вирус)
Самореплицирующийся код, заражающий файлы. Требует действий пользователя для распространения.

Worm (Червь)
Самостоятельно распространяется по сети без вмешательства пользователя.

Trojan (Троян)
Маскируется под легитимное ПО. Создаёт backdoor для удалённого управления.

Ransomware (Программа-вымогатель)
Шифрует файлы и требует выкуп за расшифровку. Примеры: WannaCry, Petya.

Spyware (Шпионское ПО)
Собирает информацию о пользователе без его ведома.

Rootkit
Скрывает присутствие вредоносного ПО в системе. Трудно обнаружить.

Phishing (Фишинг)
Мошенничество для кражи конфиденциальных данных. Поддельные письма от имени банков, соцсетей, сервисов.

Виды:
- Spear phishing - целевой фишинг на конкретного человека
- Whaling - атака на топ-менеджеров
- Smishing - фишинг через SMS
- Vishing - голосовой фишинг

Social Engineering (Социальная инженерия)
Манипуляция людьми для получения информации или доступа.

Техники:
- Pretexting - создание ложного сценария
- Baiting - приманка (флешка с вирусом)
- Quid pro quo - обмен услугами
- Tailgating - физическое проникновение следом

Man-in-the-Middle (MITM)
Перехват коммуникации между двумя сторонами. Атакующий может читать и изменять данные.

DDoS (Distributed Denial of Service)
Перегрузка сервера трафиком от множества устройств. Цель - сделать сервис недоступным.

Типы:
- Volumetric - перегрузка полосы пропускания
- Protocol - эксплуатация уязвимостей протоколов
- Application layer - атака на веб-приложение

SQL Injection
Внедрение SQL кода через пользовательский ввод. Позволяет читать, изменять, удалять данные БД.

Защита: параметризованные запросы, ORM, валидация ввода.

XSS (Cross-Site Scripting)
Внедрение JavaScript кода на веб-страницу. Кража cookies, сессий, перенаправление.

Типы:
- Stored XSS - хранится на сервере
- Reflected XSS - отражается от сервера
- DOM-based XSS - выполняется в DOM

Zero-Day эксплойты
Атаки на уязвимости, о которых разработчики ещё не знают. Нет патча.

Криптография

Шифрование
Преобразование данных в нечитаемый вид.

Симметричное шифрование
Один ключ для шифрования и расшифровки.

Алгоритмы:
- AES (Advanced Encryption Standard) - стандарт
- DES - устаревший
- 3DES - улучшенный DES
- ChaCha20 - современный stream cipher

Асимметричное шифрование
Пара ключей: публичный (для шифрования) и приватный (для расшифровки).

Алгоритмы:
- RSA - наиболее распространённый
- ECC (Elliptic Curve) - эффективнее RSA
- Diffie-Hellman - обмен ключами

Хеширование
Необратимое преобразование данных в строку фиксированной длины.

Функции:
- SHA-256 - криптографически стойкая
- SHA-3 - новый стандарт
- MD5 - устаревшая, небезопасная
- bcrypt - для паролей
- Argon2 - современный алгоритм для паролей

Цифровые подписи
Гарантируют подлинность и целостность. Используют асимметричную криптографию.

SSL/TLS
Протоколы для шифрования трафика. HTTPS использует TLS.

Сертификаты
Выдаются Certificate Authority (CA). Подтверждают владение доменом.

Аутентификация

Факторы аутентификации:
1. Что вы знаете - пароль, PIN
2. Что у вас есть - токен, смартфон
3. Кто вы - биометрия (отпечаток, лицо)

Multi-Factor Authentication (MFA)
Использование двух и более факторов. Значительно повышает безопасность.

Методы:
- SMS коды
- Authenticator apps (Google Authenticator, Authy)
- Hardware tokens (YubiKey)
- Биометрия

Безопасность паролей
Требования:
- Минимум 12 символов
- Заглавные и строчные буквы
- Цифры и специальные символы
- Уникальные для каждого сервиса

Password managers
Хранят и генерируют пароли: 1Password, LastPass, Bitwarden.

Сетевая безопасность

Firewall (Межсетевой экран)
Фильтрует входящий и исходящий трафик по правилам.

Типы:
- Packet filtering
- Stateful inspection
- Application layer
- Next-generation (NGFW)

VPN (Virtual Private Network)
Шифрованный туннель для интернет-трафика. Скрывает IP адрес.

Протоколы:
- OpenVPN
- WireGuard
- IKEv2/IPSec

IDS/IPS
- IDS (Intrusion Detection System) - обнаружение вторжений
- IPS (Intrusion Prevention System) - предотвращение вторжений

Network Segmentation
Разделение сети на сегменты для ограничения распространения атак.

Защита данных

Backup (Резервное копирование)
Правило 3-2-1:
- 3 копии данных
- 2 разных типа носителей
- 1 копия офф-сайт

Data Loss Prevention (DLP)
Мониторинг и защита от утечек данных.

Encryption at rest
Шифрование хранящихся данных.

Encryption in transit
Шифрование передаваемых данных.

Compliance и стандарты

GDPR (General Data Protection Regulation)
Европейский регламент о защите персональных данных.

PCI DSS
Стандарт для обработки платёжных карт.

HIPAA
Защита медицинских данных в США.

ISO 27001
Международный стандарт СУИБ (системы управления информационной безопасностью).

SOC 2
Аудит контролей безопасности для сервис-провайдеров.

Инциденты и реагирование

Incident Response Plan
План реагирования на инциденты:
1. Preparation - подготовка
2. Detection - обнаружение
3. Containment - локализация
4. Eradication - устранение
5. Recovery - восстановление
6. Lessons learned - выводы

Security Operations Center (SOC)
Центр мониторинга безопасности 24/7.

SIEM (Security Information and Event Management)
Сбор и анализ логов безопасности. Примеры: Splunk, ELK Stack.

Vulnerability Management
Управление уязвимостями:
- Сканирование (Nessus, OpenVAS)
- Оценка рисков
- Патчинг
- Верификация

Penetration Testing (Pentest)
Имитация атаки для проверки защиты. Этичный хакинг.

Фазы:
1. Reconnaissance - разведка
2. Scanning - сканирование
3. Gaining access - получение доступа
4. Maintaining access - сохранение доступа
5. Covering tracks - заметание следов

Bug Bounty
Программы вознаграждения за найденные уязвимости. Платформы: HackerOne, Bugcrowd.

Безопасная разработка

OWASP Top 10
Топ-10 критических уязвимостей веб-приложений:
1. Broken Access Control
2. Cryptographic Failures
3. Injection
4. Insecure Design
5. Security Misconfiguration
6. Vulnerable Components
7. Authentication Failures
8. Data Integrity Failures
9. Security Logging Failures
10. Server-Side Request Forgery

DevSecOps
Интеграция безопасности в процесс разработки. Security as Code.

Инструменты:
- SAST - статический анализ кода
- DAST - динамическое тестирование
- SCA - анализ компонентов
- Secret scanning - поиск секретов в коде

- Zero Trust Architecture
- Quantum-resistant криптография
- Blockchain для безопасности
- IoT security
